# Self-Hosted Docker Deployment

## Quick Start

```bash
cd docker
./setup.sh
docker compose up -d
```

The setup script automatically generates:
- `.env` file with random database password and JWT secret
- `keys/jwt-rsa.pem` RSA private key for JWT token signing
- `certs/aspnetapp.pfx` self-signed HTTPS certificate

## Services

| Service | URL |
|---------|-----|
| Web App (HTTP) | http://localhost:5000 |
| Web App (HTTPS) | https://localhost:5001 |
| Swagger API Docs | http://localhost:5000/swagger |
| PostgreSQL | localhost:5432 |
| pgAdmin (optional) | http://localhost:5050 |

To include pgAdmin:

```bash
docker compose --profile tools up -d
```

## Configuration

### Environment Variables

All configuration is in the `.env` file (generated by `setup.sh` from `.env.example`). Key settings:

| Variable | Description | Default |
|----------|-------------|---------|
| `DB_PASSWORD` | PostgreSQL password | Auto-generated |
| `JWT_SECRET_KEY` | Legacy JWT signing key | Auto-generated |
| `CERT_PASSWORD` | HTTPS certificate password | `password` |
| `SMTP_HOST` | SMTP server for emails | *(disabled)* |
| `SMTP_PORT` | SMTP port | `587` |
| `SMTP_USERNAME` | SMTP username | *(empty)* |
| `SMTP_PASSWORD` | SMTP password | *(empty)* |
| `SMTP_FROM_EMAIL` | Sender email address | `noreply@localhost` |

### JWT RSA Key

JWT tokens are signed with an RSA key stored at `keys/jwt-rsa.pem`. This key is generated automatically by `setup.sh` and mounted read-only into the container.

To regenerate the key (this will invalidate all existing tokens):

```bash
rm keys/jwt-rsa.pem
./setup.sh
docker compose restart web
```

### HTTPS Certificate

A self-signed certificate is generated at `certs/aspnetapp.pfx`. For production, replace this with a certificate from a trusted CA or use a reverse proxy (nginx, Caddy, Traefik) for TLS termination.

### Email (Optional)

Email is required for password reset functionality. Configure the `SMTP_*` variables in `.env`. Example for Gmail:

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USERNAME=your-email@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_ENABLE_SSL=true
SMTP_FROM_EMAIL=noreply@yourdomain.com
```

### Advanced Configuration

For additional app settings (beyond environment variables), create `config/appsettings.json` using `config/appsettings.json.example` as a template. This file is mounted into the container at `/app/config/appsettings.json`.

## Data & Backups

Persistent data is stored in Docker volumes and bind mounts:

| Data | Location |
|------|----------|
| PostgreSQL database | `postgres_data` volume |
| Uploaded files | `./uploads/` |
| Plugins | `./plugins/` |
| RSA signing key | `./keys/jwt-rsa.pem` |
| HTTPS certificate | `./certs/aspnetapp.pfx` |
| Data Protection keys | `dataprotection_keys` volume |

To back up the database:

```bash
docker exec homemanagement-db pg_dump -U homemanagement homemanagement > backup.sql
```

To restore:

```bash
docker exec -i homemanagement-db psql -U homemanagement homemanagement < backup.sql
```

## Admin CLI

The container includes an admin CLI for management tasks:

```bash
docker exec homemanagement-web admin-cli reset-password user@example.com NewPassword123
```

## Updating

```bash
docker compose pull
docker compose up -d
```

The application runs database migrations automatically on startup.

### Upgrading to RS256 JWT Signing

Versions after v1.0.0-beta11 use RS256 (asymmetric) JWT signing instead of HMAC. Existing deployments must generate a persistent RSA key or authentication will fail.

If you haven't modified your docker-compose file, re-running `setup.sh` and using the updated compose file is the easiest path. If you have a customized compose file, apply these changes manually:

1. Generate the RSA key:

```bash
mkdir -p keys
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 -out keys/jwt-rsa.pem
chmod 600 keys/jwt-rsa.pem
```

2. Add the environment variable to your `web` service:

```yaml
- JwtSettings__RsaPrivateKeyPemFile=/app/keys/jwt-rsa.pem
```

3. Add a volume mount to your `web` service:

```yaml
- ./keys:/app/keys:ro
```

4. Restart:

```bash
docker compose up -d
```

All users will need to sign in again after this change.

## Troubleshooting

### Container won't start

Check logs:

```bash
docker compose logs web
```

### JWT / authentication errors

If users get 401 errors after a restart, the RSA key may be missing or corrupted:

```bash
# Verify the key exists and is valid
openssl rsa -in keys/jwt-rsa.pem -check -noout

# If invalid, regenerate
rm keys/jwt-rsa.pem
./setup.sh
docker compose restart web
```

Note: regenerating the key invalidates all existing JWT tokens. Users will need to sign in again.

### Database connection errors

Ensure PostgreSQL is healthy:

```bash
docker compose ps
docker compose logs postgres
```
